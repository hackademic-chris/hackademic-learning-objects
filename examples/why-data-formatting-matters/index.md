# Why Data Formatting Matters: A Manager's Guide to Structured Information

## Overview

This learning object introduces data formatting as a business and digital literacy issue, not a technical trivia exercise.

The lesson is designed for non-technical executives and managers who are making decisions about AI adoption, automation, reporting, workflows, vendors, and governance. It uses JSON as the first concrete example because JSON appears frequently in AI tools, APIs, automation workflows, configuration records, logs, and evidence artifacts.

The larger point is not JSON itself. The point is that AI quality, automation quality, reporting quality, and governance quality all depend on data being structured well enough for systems and people to interpret it correctly.

Formatting is not decoration. Formatting is how information becomes usable.

This folder is the canonical source of truth for the learning object. Component files hold reusable instructional substance. Render and metadata files are derived or support artifacts.

## Learning Objective

Given a simple structured data example, the learner will explain what the format communicates, identify how formatting affects data quality or automation, and describe one business risk caused by poorly structured data.
